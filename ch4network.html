Networking and the Internet

TERMDEFINITION
Networking classifications
LAN(local area network) A collection of computers in a single building or building complex.
MAN(metropolitan area network) A network of intermediate size, such as one spanning a local community.
WANA network which links machines over a greater distanceâ€”perhaps in neighboring cities or on opposite sides of the world.
Network models
bus networkA network in which the machines are all connected to a common communication line called a bus.
star networkA network in which one machine serves as a central focal point to which all the others are connected.
Protocols
CSMA/CD EthernetThis protocol dictates that each message be broadcast to all the machines on the bus.
CSMA/CA WifiA wireless network that adopts the policy of trying to avoid collisions rather than trying to detect them.
Combining networks
repeaterA device that passes signals back and forth between the two original buses (usually with some form of amplification) without considering the meaning of the signals.
bridgeA device that connects two buses but does not necessarily pass all messages across the connection. Instead, it looks at the destination address that accompanies each message and forwards a message across the connection only when that message is destined for a computer on the other side.
switchEssentially a bridge with multiple connections, allowing it to connect several buses rather than just two.
routerA special-purpose computer used for forwarding messages.
gatewayThe "point" at which one network is linked to an internet.
Client/server model
P2P: peer-to-peerA model which involves processes that provide service to and receive service from each other.
distributed systemA system that consists of software units that execute as processes on different computers.
cloud computingA system in which huge pools of shared computers on the network can be allocated for use by clients as needed.
Internet addressing
tier 1ISPs that consist of very high-speed, high-capacity, international WANs.
tier 2ISPs that tend to be more regional in scope and less potent in their capabilities.
tier 3An ISP that is essentially an independent Internet, sometimes called an intranet, operated by a single authority that is in the business of supplying Internet access to individual homes and businesses.
IP address
ICANNA nonprofit corporation established to coordinate the Internet's operation. See ICANN.
DNS(domain name system) Essentially directories that provide address translation services to clients.
TLD: top-level domainDomain names that are descriptive of the organization registering the domain, which enhances their utility for humans.
country-code TLDsTwo-letter TLDs for each specific country. See internet country domains list.
Internet protocols
FTPAn application for listing and copying files across the network.
HTTPTraditional network applications that have come to be handled by webpages.
SMTPA way that two computers on the network may interact when transmitting an email message from one host to the other.
World Wide Web
URLA unique address to locate and retrieve documents on the Web.
client-side activitiesActivities performed by a client (such as a browser).
server-side activitiesActivities performed by a server (such as a webserver).
Internet four-layer software
application layerThe layer where a message typically originates.
transport layerThe layer where the message is sent after the application layer to be prepared for transmission.
network layerThe layer where the message is sent after the transport layer to be prepared for transmission.
link layerThe layer where the message is received at the destination and passed back up the hierarchy until it is delivered to the application layer at the message's destination.
Internet is:
packet-switching networkThe transfer of small pieces of data across various networks.
fault tolerantAn Internet that is tolerant of many types of faults.
redundantAn Internet that can fall back to other, redundant pathways for delivery when congestion or failure occur.
Cybersecurity
malwareMalicious software.
virusSoftware that infects a computer by inserting itself into programs that already reside in the machine.
wormAn autonomous program that transfers itself through a network, taking up residence in computers and forwarding copies of itself to other computers.
Trojan horseA program that enters a computer system disguised as a desirable program, such as a game or useful utility package, that is willingly imported by the victim.
spywareSoftware that collects information about activities at the computer on which it resides and reports that information back to the instigator of the attack.
phishingA technique of obtaining information explicitly by simply asking for it.
DoS attackThe process of overloading a computer with messages.
Protection
firewallA primary prevention technique which filters traffic passing through a point in the network.
spam filterFirewalls designed to block unwanted email.
proxy serverA software unit that acts as an intermediary between a client and a server with the goal of shielding the client from adverse actions of the server.
antivirus softwareSoftware used to detect and remove the presence of known viruses and other infections.
cryptographyThe study of sending and receiving messages securely in the presence of adversaries.
public key encryptionA technique by which the encryption and decryption algorithms are known, and one of the keys is known, but the encoded message remains secure.
