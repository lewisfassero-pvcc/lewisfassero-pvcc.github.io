Operating Systems Terminology

TERMDEFINITION
operating systemThe software that controls the overall operation of a computer.
job queueA storage organization in which objects (in this case, jobs) are ordered in first-in, first-out (abbreviated FIFO and pronounced "FI-foe") fashion.
FIFOOrdered in first-in, first-out (abbreviated FIFO and pronounced "FI-foe") fashion.
interactive processingTo carry on a dialogue with the user through remote terminals.
time-sharingProviding service to multiple users at the same time.
multiprogrammingThe technique in which time is divided into intervals and then the execution of each job is restricted to only one interval at a time. At the end of each interval, the current job is temporarily set aside and another is allowed to execute during the next interval. By rapidly shuffling the jobs back and forth in this manner, the illusion of several jobs executing simultaneously is created.
application softwareThe programs for performing tasks particular to the machine's utilization.
system softwareSoftware which performs those tasks that are common to computer systems in general.
utility softwarePrograms for performing activities that are fundamental to computer installations but not included in the operating system.
graphical user interface (GUI)A user interface in which objects to be manipulated, such as files and programs, are represented pictorially on the display as icons.
shellA user interface which communicated with users through textual messages using a keyboard and monitor screen.
kernelThe internal part of an operating system.
device driverThe software units that communicate with the controllers (or at times, directly with peripheral devices) to carry out operations on the peripheral devices attached to the machine.
file managerThe software unit whose job is to coordinate the use of the machine's mass storage facilities.
memory managerThe software unit which is charged with the task of coordinating the machine's use of main memory.
virtual memoryThe large "fictional" memory space created by paging.
boot strappingThe procedure that transfers the operating system from mass storage (where it is permanently stored) into main memory (which is essentially empty when the machine is first turned on).
ROMThe component which contains a collection of software routines for performing fundamental input/output activities such as receiving information from the keyboard, displaying messages on the computer screen, and reading data from mass storage.
deadlockThe condition in which two or more processes are blocked from progressing because each is waiting for a resource that is allocated to another.
spoolingA technique of holding data for output at a later but more convenient time.
super user (or administrator)A person who gains highly privileged access to the operating system by identifying him- or herself as the administrator (usually by name and password) during the login procedure.
multi-factor authenticationA solution to the pervasive problem of humans using weak passwords to secure their computer accounts which required more than one piece of evidence to gain access.
